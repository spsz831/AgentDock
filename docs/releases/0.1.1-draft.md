# AgentDock 0.1.1 Release Notes (Draft)

Status: Draft for rehearsal only.  
Date: 2026-04-09

## Summary

This patch release focuses on output stability and operational readiness:

- Unified JSON envelope for all CLI commands (`init`, `validate`, `export`, `install`, `upgrade`)
- Stable machine-readable error code set for command failures
- `upgrade` text mode changed to summary-first output, with diff behind `--verbose`
- Added end-to-end workflow tests and CI pipeline
- Added release documentation and changelog process assets

## User-Visible Changes

### 1) Unified JSON command contract

All command `--json` outputs now follow the same top-level contract:

- `schemaVersion`
- `generatedAt`
- `toolVersion`
- `command`
- `success`
- `data`
- `errors` (`[{ code, message }]`)

`upgrade` keeps its rich upgrade payload in `data` (`diff`, `summary`, warning details).

### 2) More stable CLI text output for upgrade

- Default text output now provides a concise, predictable summary.
- Full diff output is still available with `--verbose`.

## Compatibility

- Manifest compatibility: no breaking change.
  - v1 manifests remain readable and upgradable to v2.
  - v2 remains the recommended manifest version.
- JSON protocol compatibility:
  - `schemaVersion` remains `1`.
  - command outputs are now more consistent across commands.
  - existing `upgrade --json` consumers should read upgrade-specific fields from `data`.

## Validation

Planned release validation commands:

- `npm run build`
- `npm test`
- `npm run cli -- validate agentdock.yml --json`
- `npm run cli -- upgrade agentdock.yml --dry-run --json`
